# Security Policy

Naikaku AI Togo System is an automation and AI-runner workbench, so security reports matter even during MVP development.

## Please Report

Please report issues that could expose secrets, bypass runner approval gates, allow arbitrary shell execution from the browser, grant unintended desktop control, enable Git push/deploy without approval, or persist raw provider keys.

Contact: 合同会社EMYSTI via <https://www.emysti.net>

## Public Issues

Do not put API keys, tokens, private logs, screenshots with secrets, exploit payloads, or customer data in public GitHub issues. Use minimal reproduction details and redact sensitive material.

## Current Security Boundary

The public MVP is designed to demonstrate governed local automation, not unrestricted computer control. External runners must remain behind fixed presets, adapter readiness, receipts, evidence review, and artifact audit before completion is claimed.
