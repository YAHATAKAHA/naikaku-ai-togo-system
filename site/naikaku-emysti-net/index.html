AI work should pass through a cabinet.

Naikaku is a Japan-born multi-model AI workbench for governed planning, coding runners, sandboxed execution, critique, supervision, and evidence-based iteration.

Cabinet governance

Runner boundaries

Download the macOS developer preview.

The first release is intentionally honest: not a signed Mac app yet, but a downloadable package that lets technical users run the public-source workbench locally. 

tar -xzf naikaku-ai-togo-system-0.1.0-mac-dev-preview.tar.gz
cd naikaku-ai-togo-system-0.1.0-mac-dev-preview
npm ci
npm run dev

API freedom now, EMYSTI gateway later.

Use my own API keys

  • Provider, endpoint, model, and API-key alias per role.
  • Session-only secret handling through the local gateway.
  • Local replay provider for no-cost plumbing tests.

Use EMYSTI API Gateway

  • Planned official endpoint for routing, billing, audit, and model fallback.
  • Designed to sit beside BYO keys rather than replace them.
  • Future fit for teams that want fewer provider accounts and cleaner cost control.
