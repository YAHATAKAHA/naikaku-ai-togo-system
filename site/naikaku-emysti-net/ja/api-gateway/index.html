Naikaku / API Gateway

Bring your own API · EMYSTI Gateway ready

自分の API でも、将来の EMYSTI Gateway でも。

Naikaku は API をロックしません。ユーザー自身の provider key を使える設計を保ちつつ、 将来は EMYSTI API Gateway を公式入口として追加できる構造にします。

自分の API キーを使う

  • 役割ごとに provider、endpoint、model、API-key alias を設定。
  • OpenAI 互換、OpenRouter、Anthropic、Aliyun DashScope/Qwen、Gemini、ローカルモデルを想定。
  • ローカル gateway は raw secret を frontend に保存しない方針。

EMYSTI API Gateway を使う

  • モデルルーティング、コスト制御、fallback、監査ログを公式サービス化する余地。
  • 複数 provider 契約を持ちたくないユーザー向け。
  • 将来のクラウド版、チーム管理、課金と接続可能。

クラウド化するなら、ここに置く。

EMYSTI Cloud は Naikaku 本体を閉じ込めるものではなく、API ルーティング、利用量、請求、チーム設定、 runner audit、モデル fallback を扱う control plane として設計するのが自然です。
