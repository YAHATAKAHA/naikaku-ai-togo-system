Naikaku / ランナー接続

Mac · CLI ランナー · 明示承認

既存ツールを、監督付き runner として扱う。

Naikaku は Codex CLI、Claude Code、OpenHands、OpenClaw 系ツール、Hammerspoon などを 直接置き換えるものではありません。既存の CLI やローカル自動化を、権限確認、実行記録、証拠レビューの後ろに置くための統合層です。

実行前の準備確認

runner の種類、作業ディレクトリ、許可されたコマンド、ネットワーク利用、成果物の置き場所を先に確認します。

実行ログと証拠

コマンド、終了コード、出力要約、変更ファイル、スクリーンショットなどを記録し、後からレビューできる形にします。

危険操作は明示承認

本番 deploy、Git push、外部送信、購入、広い PC 操作は自動で通さず、承認ゲートと根拠を求めます。

接続対象の考え方。

外部プロジェクトを勝手に取り込むのではなく、各ツールのライセンスと使い方を尊重します。 Naikaku 側は runner adapter、権限プリセット、証拠レビュー、内閣ロールの判断ログを担当します。

adapter を書く

コマンドラインツールを Naikaku の実行契約に合わせるための貢献入口。

モデルを分ける

監督役、批評役、実行役に別 provider や別 model を割り当てる考え方。

ローカルで試す

まずは開発者向け preview を落として、証拠付きの smoke test から始めます。
