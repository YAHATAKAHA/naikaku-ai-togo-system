# Architecture Decision Records

This directory stores accepted architecture and product-direction decisions that should remain visible during public-source review.

## Records

- [ADR-0001: Use Five Strategy Iterations as a Public-Source Alignment Gate](./0001-strategy-iteration-gate.md)

## Policy

Add an ADR when a change affects Naikaku's long-lived direction, including cabinet governance, role data boundaries, runner permission models, deployment posture, licensing boundaries, or public-source contribution rules.

Small implementation details do not need an ADR. They should stay in code comments, tests, or the related feature documentation.
